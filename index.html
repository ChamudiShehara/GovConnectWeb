About GovConnect

Automating Government–Citizen Complaint Management

GovConnect is a research-driven web application designed to bridge the communication gap between citizens and government. Citizens submit complaints through the platform, which are then automatically classified to the relevant government department using a fine-tuned Ollama LLaMA 3 model and a RAG pipeline that retrieves and categorizes department data.

Once posted to the department, complaints are prioritized by urgency using a RandomForestRegressor ML model that considers citizen votes, officer perspectives (High / Medium / Low), and keyword analysis. The department minister reviews and assigns agents who update complaint statuses through to resolution.

  • ✓ AI-driven department classification via RAG + LLaMA 3 (Ollama)
  • ✓ ML-based urgency scoring using RandomForestRegressor
  • ✓ Citizen voting & officer priority input (High / Medium / Low)
  • ✓ Minister dashboard with agent assignment workflow
  • ✓ Full MERN stack + Vite + TailwindCSS v4 architecture

— Research Components —
🧠

Component 1 — Department Classification

Automatically detects the relevant government department for each citizen complaint using a fine-tuned Ollama LLaMA 3 model. The RAG pipeline retrieves government department data, cleans and categorizes it, and performs semantic matching to route complaints accurately.

Ollama LLaMA 3 RAG Pipeline JavaScript
📊

Component 2 — Urgency Prioritization

Prioritizes complaints using a RandomForestRegressor ML model. Considers citizen votes, officer priority perspective (High / Medium / Low), and keyword analysis to produce an urgency score. The model is trained on a labeled dataset to predict complaint priority.

RandomForestRegressor scikit-learn Python Pandas

AI Classification

RAG retrieves department data and suggests the relevant department using semantic matching. LLaMA 3 handles ambiguous cases as a fallback.

Citizens can submit complaints through an intuitive form with auto-suggested department routing powered by AI.

How It Works

Complaint Resolution Pipeline

From submission to resolution — an AI-augmented workflow that ensures every complaint is handled efficiently.

01

Citizen Submits

A citizen fills out the complaint form with title, description, and optional department preference.

02

AI Classifies

RAG + LLaMA 3 analyze the complaint text and automatically route it to the correct government department.

03

Minister Reviews

The department minister reviews incoming complaints and assigns agents for investigation and resolution.

04

Agent Resolves

Assigned agents work on complaints, update statuses, and mark them resolved — with full audit trail.
